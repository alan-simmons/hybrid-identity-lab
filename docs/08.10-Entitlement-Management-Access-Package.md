# 🔐 Phase 08.10 — Entitlement Management (Access Packages)

## 🎯 Objective

Implement automated access provisioning using Microsoft Entra ID Entitlement Management.

This lab simulates a real-world onboarding workflow where users can request access to resources, receive approval, and have access granted automatically.

---

## 🧱 Architecture Overview

This access package includes:

- **Cloud-Users** (Security Group)
- **Lab-Test-App** (Enterprise Application)

Provisioning flow:


User requests access
↓
Approval required
↓
Access granted
↓
User added to group + assigned to application


---

## ⚙️ Configuration Steps

### 1. Create Access Package

Navigate to:

- Microsoft Entra Admin Center
- Identity Governance
- Access packages
- Create new access package

**Configuration:**

- Name: `New Employee Package`
- Description: Provides baseline access for new employees
- Catalog: General

---

### 2. Add Resources

Resources added:

- Cloud-Users → Role: Member
- Lab-Test-App → Role: User

![Resources Added](../screenshots/08.9/1.png)

---

### 3. Configure Request Settings

Who can request:

- All members (excluding guests)

Who can request access:

- Self
- Admin

Approval:

- Require approval: Yes
- Require justification: Yes
- Approval stages: 1

![Request Settings](../screenshots/08.9/2.png)

---

### 4. Configure Approvers

Approver configuration:

- Type: Selected users or groups
- Approvers:
  - Cloud-Admins
  - Alan Simmons

Decision settings:

- Decision timeframe: 14 days
- Require approver justification: Yes

This ensures redundancy and avoids single-user dependency.

---

### 5. Configure Lifecycle

Access expiration:

- Expiration type: Number of days
- Duration: 30 days
- User-defined duration: Disabled

Access reviews:

- Disabled (handled separately in Phase 08.9)

This enforces time-bound access control.

## 🧠 Design Decisions

### Why use an Access Package?

Access packages allow:

- Centralized access control
- Self-service requests
- Automated provisioning
- Approval workflows

---

### Why include both group and application?

This simulates real onboarding:

- Group membership controls general access
- Application assignment grants service access

---

### Why use multiple approvers?

Using both:

- Cloud-Admins group
- Direct user (Alan Simmons)

Ensures:

- High availability of approvals
- No dependency on a single identity

---

### Why enforce expiration?

Time-bound access ensures:

- Least privilege over time
- Automatic cleanup of stale access
- Reduced security risk
