# Phase 08.9 — Access Reviews

## Objective
Implement Microsoft Entra Access Reviews to periodically validate membership in a privileged cloud administration group.

This lab demonstrates identity governance by reviewing access to the `Cloud-Admins` group on a recurring schedule. The review is assigned to a dedicated governance team rather than an individual user to ensure continuity if personnel change.

---

## Why This Matters

Access reviews help prevent **privilege creep**, where users retain access longer than necessary.

This is a core identity governance control used to support:

- Least privilege
- Zero Trust
- Periodic access certification
- Compliance and audit readiness

---

## Lab Design

### Reviewed Resource

- **Group:** `Cloud-Admins`

### Reviewer Model

- **Reviewer Group:** `Cloud-Governance`

### Review Behavior

- **Recurrence:** Quarterly  
- **Duration:** 7 days  
- **Scope:** All users  
- **Auto apply results:** Enabled  
- **If reviewers do not respond:** Remove access  

---

## Architecture

The governance workflow for this lab is:

```text
User added to Cloud-Admins
        ↓
Quarterly access review triggered
        ↓
Cloud-Governance reviewers evaluate membership
        ↓
Approve or deny access
        ↓
Results automatically applied

This design separates privileged access from governance oversight, which reflects real-world IAM architecture.

## Configuration Steps

### 1. Create the Access Review

In **Microsoft Entra Admin Center**:

1. Go to **Identity Governance**
2. Select **Access Reviews**
3. Create a new review
4. Choose **Teams + Groups**
5. Select **Cloud-Admins**
6. Scope the review to **All users**

![Access Review Scope](../screenshots/08.9/access-reviews/review-scope.png)

---

### 2. Assign Reviewers and Recurrence

The review was configured to use a dedicated governance group.

**Reviewer Configuration**

- **Reviewer type:** Selected user(s) or group(s)
- **Reviewer group:** Cloud-Governance
- **Duration:** 7 days
- **Review recurrence:** Quarterly
- **End condition:** Never

Using a governance group instead of a single reviewer ensures that the review process continues even if a specific team member leaves or changes roles.

![Reviewers and Recurrence Settings](../screenshots/08.9/access-reviews/reviewers-recurrence.png)

---

### 3. Configure Completion and Enforcement Settings

The review was configured with automatic governance enforcement.

**Settings**

- **Auto apply results to resource:** Enabled
- **If reviewers don't respond:** Remove access
- **No sign-in within 30 days:** Enabled
- **Justification required:** Enabled
- **Email notifications:** Enabled
- **Reminders:** Enabled

This configuration simulates a security-first governance process where stale or unreviewed privileged access is automatically removed.

![Completion Settings](../screenshots/08.9/access-reviews/completion-settings.png)
